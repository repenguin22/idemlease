## レビュー原文

## 1. 総評と Top 3 リスク

コアのリース状態機械（token CAS・原子 Reserve・`LeaseLost` 正規化）と memstore / Lua 1-key スクリプトは、**時計が正常な前提では「有効リース保持者が高々 1」を破る実行順序は見つからなかった**。`go test -race ./...`（ルート）は通過。Redis 未起動のため redistore はコードレビューのみ。

一方、**リプレイの HTTP セマンティクス**と **redistore の「存在＝ Redis EXISTS」対「論理期限＝アプリ絶対時刻」**に、契約・実運用で刺さる穴がある。Critical はなし。v1.0.1 では High/Medium を潰すべき。

**Top 3 リスク**
1. **手動 gzip 等の圧縮レスポンスが、リプレイで壊れる**（ボディは圧縮のまま、`Content-Encoding` は非保存）
2. **redistore がアプリ時計の論理期限を Reserve で強制しない** → 時計ずれ時に期限切れリプレイ / sticky 409
3. **`Begin` が `existing` に対し `Expired` を見ない** → Store 契約違反や (2) を吸収できず、誤判定のまま進む

### High

**H1. 圧縮ボディのサイレント破壊（リプレイ）** — `httpidem/recorder.go` / `httpidem/httpidem.go` — §4.4
ハンドラが gzip バイトを書き、`Content-Encoding: gzip` を付与。初回はクライアントが解凍できる。Persist 時 allowlist 外のため `Content-Encoding` は捨てられ、圧縮ボディだけ保存。リトライの Replay は生 gzip を「非圧縮」として返す → パース失敗・文字化け。

### Medium

**M1. redistore: 論理期限と EXISTS の乖離（期限切れリプレイ / sticky 409）** — `redistore/redistore.go` — §3.1, §3.2
アプリ時計が前にジャンプ / 他ノードが進んでいる場合、`rec_exp_ms`/`lease_exp_ms`（アプリ絶対時刻）と Redis TTL が乖離し、論理期限超過後も EXISTS の間 `Replay`/`RejectInFlight`（sticky 409）になる。修正案: PTTL を正とする。

**M2. `Begin` が `existing.Expired` を見ない（防御欠如）** — `idemlease.go` — §3.1
Store が期限切れを `ErrAlreadyExists` で返した場合に誤 409 / 誤 Replay。修正案: 契約違反エラー、または Reserve 再試行。

**M3. リプレイ経路にペイロードサイズ上限がない** — `httpidem/httpidem.go` / `storedresponse.go` — §6.1
Store に巨大ペイロードを載せられると Replay で無制限アロケーション。修正案: 上限チェック + デコーダ件数上限。

**M4. 適合性スイートが H1 / M1 を固定していない** — §12-5, §12-7

**M5. リクエスト Body 差し替え時に元 `Body.Close()` していない** — `httpidem/httpidem.go`

**M6. 契約文面と Flusher 実装の食い違い** — REQUIREMENTS §6.1 vs `httpidem/recorder.go`（契約ドリフトとして記録推奨）

### Low

**L1.** `r.ContentLength` をバッファ後に更新しない。**L2.** デフォルト allowlist に `Content-Disposition` がない。**L3.** ヘッダ依存ボディ全般のポリシー明文化。**L4.** Complete 失敗ログの過剰警告（→ 検証の結果、二重警告は発生しないことを確認・見送り）。**L5.** Recorder 非スレッドセーフの README 記載。

### Nit

**N1.** `Methods` は upper 登録だが `r.Method` は生比較。**N2.** 単一ヘッダ内の `a, b` は生キー扱い（§4.1 どおり）。**N3.** `bytes.Equal` の非定数時間は実害なし。

### 確認済みで問題なしの領域

状態機械（判定表・token CAS・LeaseLost 正規化）、redistore Lua の原子性・1-key クラスタ制約、キー文法・指紋・捕捉の基本、KeyScope の NUL 分解一意性、5xx detail 非露出、並行スイート、API 設計、README の必須記載。
